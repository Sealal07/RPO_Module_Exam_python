# Отчет об устранении инцидентов в системе NexusMonitor

## Инцидент No1: Конфликт версии SQLAlchemy с Python 3.13
──────────────────────────────────────────────────
СИМПТОМ:
   При запуске uvicorn падает AssertionError из недр SQLAlchemy: "Class SQLCoreOperations directly inherits TypingOnly but has additional attributes".

ПРИЧИНА:
   В requirements.txt зафиксирована старая версия sqlalchemy==2.0.25, которая несовместима с изменениями в модуле typing в свежих версиях Python (3.13+). Также зависимости устанавливались глобально, минуя изолированное окружение.

РЕШЕНИЕ:
   Активировано виртуальное окружение venv, в файле backend/requirements.txt версия sqlalchemy обновлена до 2.0.31, выполнена повторная установка через pip install.

## Инцидент No2: Отсутствие пакета pydantic-settings в зависимостях
──────────────────────────────────────────────────
СИМПТОМ:
   Бэкенд падает при запуске с ошибкой: "ModuleNotFoundError: No module named 'pydantic_settings'" при инициализации core/config.py.

ПРИЧИНА:
   В коде конфигурации используется класс BaseSettings из библиотеки pydantic_settings, однако сам пакет не был добавлен стажером в файл манифеста зависимостей requirements.txt.

РЕШЕНИЕ:
   В файл backend/requirements.txt добавлена строка 'pydantic-settings==2.1.0', выполнена сборка зависимостей через pip install.

## Инцидент No3: Отсутствие совместимого драйвера PostgreSQL для Python 3.13
──────────────────────────────────────────────────
СИМПТОМ:
   Uvicorn падает с "ModuleNotFoundError: No module named 'psycopg2'". При попытке установить psycopg2-binary==2.9.9 сборка падает с ошибкой "Error: pg_config executable not found".

ПРИЧИНА:
   Для работы SQLAlchemy с PostgreSQL необходим драйвер. Версия 2.9.9 не имеет готовых wheel-пакетов под архитектуру Python 3.13 на Windows, из-за чего pip пытается собрать пакет из исходников, требуя установленный в системе pg_config.

РЕШЕНИЕ:
   В backend/requirements.txt версия пакета заменена на psycopg2-binary==2.9.10, поставляющуюся с готовыми скомпилированными бинарными файлами под Python 3.13.

## Инцидент No4: Отсутствие дефолтных настроек БД и неверный порт в конфигурации
──────────────────────────────────────────────────
СИМПТОМ:
   Бэкенд падал с ошибкой OperationalError (Connection refused) при попытке подключения к порту 5433. При отсутствии переменных в .env класс Settings выбрасывал ошибку валидации.

ПРИЧИНА:
   В core/config.py поля класса Settings (BaseSettings) не имели значений по умолчанию и требовали обязательного наличия переменных в файле .env. При этом дефолтный порт СУБД в системе был ошибочно указан стажером как 5433 вместо стандартного порта PostgreSQL (5432).

РЕШЕНИЕ:
   В файле backend/core/config.py для всех полей класса Settings заданы безопасные значения по умолчанию, исправлен порт DB_PORT на стандартный 5432.
